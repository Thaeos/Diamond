## 10) Entry Points (Common Runs)

**Single command to bring everything together:** `npm run together` ‚Äî see **TOGETHER.md**.

From `package.json`:

- `npm run together` ‚Üí full flow (env ‚Üí RPCs ‚Üí test:systems ‚Üí Tenderly ‚Üí repos ‚Üí check-diamond-rpc ‚Üí summary)
- `npm run proceed` ‚Üí quick preflight (test:systems, treasure-repos-check, tenderly status)
- `npm run setup` ‚Üí `setup_monitoring.sh`
- `npm run monitor` ‚Üí `start_monitoring.sh`
- `npm run check-diamond-rpc`
- `npm run tenderly-diamond` / `npm run tenderly-diamond-repos`
- `npm run blockscout-monitor`
- `npm run deploy-diamond`
- `npm run fetch-rpcs`
- **`npm run agent -- <inquiry>`** ‚Äî Autonomous agent entry: status, scope, balance, propose. Executes within the scope of your inquiry; Safe writes require your approval. See **AGENT.md**.

---

## 11) Suggested Run Order (When Starting Fresh)

**Or run once:** `npm run together` (does steps 1‚Äì4 below and reports next steps).

1) Configure creds in `env.txt`
2) Add addresses to `diamond_deployments.json`
3) Fetch RPC pools: `npm run fetch-rpcs -- <chainIds>` (or let `together` do it if chainlist missing)
4) Verify with `npm run check-diamond-rpc`
5) Run `npm run setup`
6) Start monitoring: `npm run monitor`
